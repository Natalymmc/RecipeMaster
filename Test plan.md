## Введение

Документ описывает план тестирования кулинарного веб‑приложения **Recipemaster**. Цель тестирования – проверить соответствие фактического поведения реализованных функций требованиям и ожиданиям, выявить дефекты до развёртывания.

Основание: анализ исходного кода фронтенда (React + TypeScript: `AuthPage.js`, `LoginPage.js`, `RegisterPage.js`, `HomePage.js`, `RecipePage.js`, `RecipeForm.js`) и бэкенда (Spring Boot: `AuthController`, `serController`, `RecipeController`).

Ключевой функционал в зоне тестирования: регистрация/авторизация, гостевой режим, просмотр/поиск рецептов, детальный просмотр рецепта, добавление рецепта с ингредиентами и шагами, добавление рецепта в избранную коллекцию.

<a name="items"/>

## Объект тестирования

**Фронтенд:** Single Page Application на JavaScript + Vite:
- Authpage - начальная страница
- LoginPage - форма входа
- RegisterPage - форма регистрации  
- Playlists - управление плейлистами
- HomePage - главная страница с рецептами и поиском
- SearchPage - расширенный поиск и фильтры
- ResipePage - детальный просмотр рецепта
- RecipeForm - Форма создания рецепта

**Бэкенд:** Spring Boot REST API.

Основные REST‑эндпоинты:

- `/api/auth/register — регистрация пользователя
- `/api/auth/login — вход в систему
- `/api/users — управление пользователями (GET все)
- `/api/recipes — CRUD рецептов 
- `/api/recipes/search — поиск рецептов по запросу, категории, сложности и сортировке
- `/api/recipes/recent — получение недавних рецептов
- `/api/recipes/{recipeId}/favorites — добавление/удаление в избранное
- `/api/recipes/favorites — получение избранных рецептов пользователя
- `/api/recipes/* — управление рецептами и избранным
- `/api/backup` — резервное копирование
- `/api/search` — поиск по всей библиотеке

<a name="quality"/>

## Атрибуты качества

1. **Функциональность**
   - Полнота: реализованы все заявленные функции (создание/просмотр рецептов, поиск, управление избранным, пользователи)
   - Корректность: API возвращает ожидаемые коды и данные; фронтенд корректно отображает состояние (например, динамическая загрузка избранных и недавних рецептов)

2. **Удобство использования**
   - Интерфейс интуитивен: понятная навигация, приятный визуал
   - Базовые операции (поиск, добавление в избранное, создание рецепта) выполняются за ≤ 3 шага

3. **Надёжность**
   - Целостность данных при валидных и невалидных операциях (валидация DTO, обработка ошибок в контроллерах)

4. **Производительность**
   - Время загрузки списка рецептов ≤ 2 секунд
   - Плавная работа форм с множественными ингредиентами/шагами

5. **Совместимость**
   - Работа в современных браузерах (Chrome, Firefox, Safari, Edge)
   - Корректное отображение на разных разрешениях экрана

<a name="risk"/>

## Риски

- Проблемы с загрузкой изображение в разных браузерах
- Большие объемы данных при поиске могут замедлять работу
- Проблемы с JWT токенами при обновлении страницы

<a name="nonfunctional"/>

## Нефункциональные требования

- Время отклика интерфейса ≤ 100 мс
- Время загрузки изображений, списков ≤ 2 секунд
- Поддержка форматов изображение: JPEG, PNG, GIF
- Максимальный размер изображения : 5 МВ
- Совместимость с современными браузерами
- Адаптивный дизайн для мобильных устройств

<a name="approach"/>

## Подходы к тестированию

- **Ручное функциональное тестирование** фронтенда через браузер
- **API тестирование** с помощью Postman ((проверка JWT, DTO валидации))
- **Юзабилити тестирование** интерфейса
- **Кросс-браузерное тестирование**
- **Тестирование производительности загрузки списков и форм

**Инструменты:**
- Postman для API тестирования
- Chrome DevTools для отладки
- Lighthouse для оценки производительности
- Ручное тестирование на разных устройствах
